# Unity Mobile Code Samples

Unity 기반 모바일 키우기 게임을 개발·출시·운영하면서 사용했던 코드 구조를 포트폴리오 제출용으로 정리한 샘플 저장소입니다.

전체 상용 프로젝트를 공개하는 대신, 실제 라이브 서비스 과정에서 중요하게 다뤘던 인앱결제, 광고 로드, 저장, 숫자 포맷, 오브젝트 풀링 관련 코드를 일부 샘플 형태로 정리했습니다.

실제 프로젝트의 API Key, 광고 ID, 상품 ID, 서버 정보, 비공개 리소스, 세부 보상량 등은 제거하거나 예시값으로 변경했습니다.

---

## 주요 코드

### 1. IAPV5Manager.cs

Unity IAP V5 기반 인앱결제 처리 코드입니다.

- 상품 등록 및 구매 가능 상태 확인
- 구매 요청 전 App Store / Google Play 상품 상태 체크
- 구매 성공 / 실패 / 보류 상태 처리
- 구매 완료 후 보상 지급 함수 연결
- 중복 보상 지급 및 미지급 가능성 방지
- 영수증 기반 중복 보상 지급 방지
- 보상 지급 후 구매 확정 처리

실제 라이브 서비스 과정에서 인앱결제 오류, 보상 미지급, 자동 환불 가능성 등을 겪으며 안정성을 높이기 위해 개선한 코드입니다.

---

### 2. IapRewardExample.cs

인앱결제 성공 이후 실제 게임 보상을 지급하는 예시 코드입니다.

- 상품 ID 기반 보상 분기
- 다이아, 패키지, 광고 제거 등 상품 유형별 처리
- 구매 후 재화 및 구매 상태 저장
- 구매 성공 UI 호출
- 상점 상태 갱신 흐름 예시

전체 프로젝트 코드를 공개하지 않고도 인앱결제 이후 보상 지급 구조를 이해할 수 있도록 실제 흐름을 단순화해 정리했습니다.

---

### 3. RewardedAdManagerExample.cs

AdMob 보상형 광고 로드와 보상 지급 흐름을 정리한 샘플 코드입니다.

- 플랫폼별 광고 ID 분기
- 테스트 광고 / 실제 광고 전환
- 광고 로드 실패 시 재시도
- 광고 준비 전 로딩 UI 처리
- 광고 시청 완료 후 보상 지급
- 광고 실패, 네트워크 미연결, 응답 지연 상황 처리
- 광고 시청 후 쿨타임 적용
- 광고 객체 Destroy 및 재로드 처리

실제 라이브 서비스 과정에서 광고 로드 실패, 응답 지연, 보상 미지급 가능성을 줄이기 위해 사용했던 흐름을 포트폴리오용으로 정리했습니다.

---

### 4. SaveDataExample.cs

모바일 키우기 게임에서 저장해야 하는 주요 데이터를 정리한 예시 클래스입니다.

- 재화 데이터
- 성장 데이터
- 광고 / 결제 상태
- 오프라인 보상 상태
- 환경설정 데이터
- 저장 버전 관리
- 기존 유저 데이터 보정 처리

실제 프로젝트에서는 더 많은 데이터가 있었지만, 저장 구조와 데이터 마이그레이션 흐름을 보여주기 위해 핵심 항목만 샘플로 분리했습니다.

---

### 5. LocalAndBackendSaveExample.cs

로컬 저장과 뒤끝 Backend 기반 클라우드 저장 흐름을 정리한 샘플 코드입니다.

- PlayerPrefs 기반 로컬 저장 / 불러오기
- JSON 직렬화 / 역직렬화
- 뒤끝 CloudSave 업로드 / 다운로드 흐름
- 클라우드 불러오기 후 로컬 저장 갱신
- 네트워크 미연결 처리
- 저장 / 불러오기 쿨타임 처리
- 기존 유저 데이터 보정

실제 라이브 서비스 과정에서 기기 저장과 서버 저장의 데이터 흐름을 맞추고, 업데이트 이후 기존 유저 데이터가 깨지지 않도록 관리했던 경험을 바탕으로 정리했습니다.

---

### 6. NumberFormatter.cs

키우기 / 방치형 게임에서 커지는 재화 수치를 짧게 표기하기 위한 숫자 포맷 유틸입니다.

- 큰 수를 한국식 단위로 변환
- 만, 억, 조, 경, 해 등 단위 처리
- 재화 / 수익 / 업적 조건별 표기 분리
- 불필요한 소수점 제거
- 음수, NaN, Infinity 예외 처리
- 상세 단위 표기 지원

방치형 게임은 숫자가 빠르게 커지기 때문에, 유저가 현재 성장 상태를 직관적으로 이해할 수 있도록 UI에 맞는 숫자 표기 처리가 중요하다고 생각해 정리한 코드입니다.

---

### 7. TouchEffect Object Pooling

터치 이펙트처럼 짧은 시간에 반복 생성되는 UI 효과를 매번 생성 / 삭제하지 않고 재사용하기 위한 오브젝트 풀링 예시입니다.

- 시작 시 이펙트 오브젝트를 미리 생성
- 터치 시 비활성화된 오브젝트를 재사용
- 재생 후 FadeOut 처리
- 완료 후 SetActive(false)로 풀에 반환
- 반복 입력 상황에서 Instantiate / Destroy 사용을 줄여 GC 부담 완화

작은 기능이지만 모바일 환경에서 반복 입력과 UI 이펙트가 많아질 때 성능 저하를 줄이기 위해 적용한 구조입니다.
